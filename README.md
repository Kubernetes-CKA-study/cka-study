## Convention
- 기본적으로 문서를 업로드할 때는 새로운 브랜치를 분기한다.
- 브랜치명은 `week01/nickname` 등 해당 주차와 작성자를 확인할 수 있는 이름으로 만든다.
- 해당 주차 폴더 아래에 본인 이름이나 닉네임으로 새로운 폴더를 만든다. (e.g. `week01/pearl`)
- 폴더 안에 md 파일이나 코드, 참고 img 경로를 모두 넣어 다른 사람의 경로와 겹치지 않게 관리한다.
- PR 형식은 [PR template](.github/PULL_REQUEST_TEMPLATE.md)을 참고한다.



## Rules
- 주 1회 오프라인 모임 진행
- 스터디 내용은 개념(이론) + 실습(커맨드, docs 활용)
- 랜덤 초기 발제자(2명 내외)를 뽑아 해당 주차 정리한 내용을 메인 발표로 진행한다.
- 발표 후, 나머지 스터디원들이 첨언하고 싶은 부분을 나누고 서로 토론하는 형식으로 진행한다.
